\section{Methods}
We implemented a reproducible benchmark pipeline using the project's experiments modules (notably \showresfile{experiments/run_canonical_experiment.py} and \showresfile{experiments/run_b2_batched.py}). The codebase generates per-iteration metadata files under \showresfile{results/} that fully describe the dataset, graph construction, method, hyper-parameters and evaluation fold used to produce each reported metric.

Datasets and preprocessing: The canonical pack includes synthetic generators (alpha, beta, broad) and real/proxy datasets (\texttt{physionet\_eegmmidb}, BCI IV 2a proxies). Synthetic generators create deterministic test signals and spatial positions (see \showresfile{experiments/run_canonical_experiment.py} for exact routines). Real-data loading follows the local dataset policy and may subsample channels for memory efficiency when necessary (see the script for details).

Graph construction and families: graph constructors explored include k-NN variants (k=3,5), variable-k schemes (vknng), Gaussian kernels, NN-K, AEW and Kalofolias learning. Reconstruction methods are grouped into two families: instant/position-based interpolators (e.g., mean, nearest, spherical spline, RBF interpolants, tikhonov/GSP variants) and TV/time-aware methods (graph–time tikhonov, TV, TRSS and temporal-laplacian). Exact parameter grids used during inner validation are defined in the experiment scripts and reproduced in the metadata.

Validation and metrics: experiments use a nested validation protocol (inner fold for parameter selection, outer fold for reporting). The pipeline uses an inner fraction of 60% (parameter search) and reports results on the remaining 40% (outer fold). Primary metrics are MAE (primary), RMSE, DTW and SNR; statistical tests and family-level significance are computed and stored in \showresfile{results/} (for example \showresfile{results/b3_stat02_significance.csv}).

Reproducibility and traceability: every numerical claim in this paper is linked to a results artifact (see claim traceability tables). When MATLAB/GSPBox comparisons were reproduced in Python, those runs are explicitly annotated as proxy validations; the script \showresfile{experiments/ins13_strict_matlab_compare.py} summarizes cross-stack checks and the canonical INS-13 status file records the editorial decision.

Engine and validation notes: experiments were executed with the pipeline engine versions v6–v7e depending on the phase; per-pack metadata records the engine and seeds used (for example \showresfile{results/it100_final_comparative_publication_pack_run_metadata.json}). The pipeline uses nested parameter selection (inner search) and outer reporting folds; all per-iteration CSVs include the full parameter set and seeds so results can be reproduced exactly.

Artifact types produced per iteration include: raw results CSV, aggregated stats CSV, significance CSV, QA report (Markdown), run metadata JSON and up to 11 figure PDFs (fig01–fig11). These artifacts are the primary sources for tables and figures in this manuscript and are linked directly in the claim traceability tables.
